Module: Monitor App Performance with Application Insights

Giám sát và phân tích hiệu suất ứng dụng với Azure Monitor Application Insights

🔷 1. Application Insights — Tổng Quan

Application Insights là extension của Azure Monitor, cung cấp tính năng Application Performance Monitoring (APM). APM tools giúp monitor applications từ development, qua testing, đến production:

  • Chủ động (Proactively) — Hiểu ứng dụng đang hoạt động như thế nào
  • Phản ứng (Reactively) — Review dữ liệu thực thi để xác định nguyên nhân sự cố

Ngoài thu thập metrics và telemetry, Application Insights còn có thể thu thập và lưu trữ application trace logging data.

Your Application
Web App / API / Functions / Mobile
App Insights SDK / Agent
Sends telemetry automatically
Azure Monitor — Application Insights
Metrics — request rates, durations
Logs — trace & diagnostic data
Traces — distributed transaction flow
Exceptions — with stack traces
Dashboards, Alerts, Workbooks

🔷 2. Các Tính Năng Chính

📡 Live Metrics

Quan sát activity từ app đang chạy theo real-time, không ảnh hưởng host environment.

🌐 Availability

Synthetic Transaction Monitoring — probe external endpoints để test availability và responsiveness theo thời gian.

🔗 GitHub/DevOps Integration

Tạo GitHub hoặc Azure DevOps work items trong context của Application Insights data.

👥 Usage Analytics

Hiểu tính năng nào phổ biến, cách users tương tác và sử dụng ứng dụng.

🤖 Smart Detection

Tự động phát hiện failures và anomalies qua proactive telemetry analysis.

🗺️ Application Map

Top-down view kiến trúc ứng dụng với visual references về health và responsiveness của từng component.

🔍 Distributed Tracing

Search và visualize end-to-end flow của một execution hoặc transaction cụ thể.

🚨 Alerts

Cấu hình alerts dựa trên metrics, logs hoặc availability test results. Thông báo qua email, SMS, webhooks.

🔹 Instrument ASP.NET Core App

CLI
dotnet add package Microsoft.ApplicationInsights.AspNetCore
C#
// Program.cs
builder.Services.AddApplicationInsightsTelemetry(
    builder.Configuration["ApplicationInsights:InstrumentationKey"]
);
JSON
// appsettings.json
{
  "ApplicationInsights": {
    "InstrumentationKey": "your-instrumentation-key",
    "ConnectionString": "InstrumentationKey=xxx;IngestionEndpoint=https://..."
  }
}
⚠️ Connection String vs Instrumentation Key

Microsoft khuyến nghị dùng Connection String thay vì Instrumentation Key. Connection String bao gồm Ingestion Endpoint để gửi telemetry đến đúng endpoint (quan trọng với sovereign clouds và custom endpoints).

🔹 Cấu Hình Availability Test

  • Chọn ít nhất 5 locations để có kết quả tin cậy
  • Đặt Success criteria: response code mong đợi, timeout
  • Cấu hình alerts khi test fails (bao nhiêu locations fail → alert)
  • Dùng dependent requests để cũng load resources (images, CSS)
C#
// Custom availability test với TrackAvailability()
public class CustomAvailabilityTest
{
    private readonly TelemetryClient _telemetry;

    public async Task RunTest()
    {
        var availability = new AvailabilityTelemetry
        {
            Name = "MyCustomTest",
            RunLocation = "East US",
            Success = false
        };

        var stopwatch = Stopwatch.StartNew();
        try
        {
            // Thực hiện test logic
            var result = await RunTestLogicAsync();
            availability.Success = result;
        }
        catch (Exception ex)
        {
            availability.Message = ex.Message;
            _telemetry.TrackException(ex);
        }
        finally
        {
            stopwatch.Stop();
            availability.Duration = stopwatch.Elapsed;
            availability.Timestamp = DateTimeOffset.UtcNow;
            _telemetry.TrackAvailability(availability);
        }
    }
}

🔷 6. Application Map

Application Map là top-down view của kiến trúc ứng dụng — bạn có thể thấy tất cả components, dependencies và health status của chúng trong một sơ đồ.

✅ Healthy Components
API Gateway — 1ms avg response
Auth Service — healthy, low latency
Orders API — 2ms avg response
Orders DB — 5ms avg, no errors
⚠️ Degraded Components
Payment API — 500ms avg, high latency warning
Payment Gateway (3rd party) — external dependency, slow upstream
Color coding: ✅ <5% failure rate  |  ⚠️ Warning  |  ❌ High failure rate

Dùng Application Map để:

  • Nhanh chóng xác định bottlenecks và failure points
  • Xem dependency health (external APIs, databases)
  • Drill down vào component cụ thể để xem chi tiết
  • Identify slow external dependencies

🔷 8. Code Examples — Track Custom Events & Metrics

C#
// Inject TelemetryClient
public class OrderController : Controller
{
    private readonly TelemetryClient _telemetry;

    public OrderController(TelemetryClient telemetry)
        => _telemetry = telemetry;

    public async Task<IActionResult> PlaceOrder(Order order)
    {
        // Track custom event
        _telemetry.TrackEvent("OrderPlaced", new Dictionary<string, string> {
            ["OrderId"]   = order.Id,
            ["UserId"]    = order.UserId,
            ["Total"]     = order.Total.ToString()
        });

        // Track custom metric
        _telemetry.TrackMetric("OrderValue", order.Total);

        // Track dependency (external call)
        var startTime = DateTime.UtcNow;
        var timer = Stopwatch.StartNew();
        try {
            await _paymentService.ProcessAsync(order);
            _telemetry.TrackDependency(
                "HTTP", "PaymentGateway", "ProcessPayment",
                startTime, timer.Elapsed, success: true
            );
        }
        catch (Exception ex) {
            _telemetry.TrackException(ex, new Dictionary<string, string> {
                ["OrderId"] = order.Id
            });
            _telemetry.TrackDependency(
                "HTTP", "PaymentGateway", "ProcessPayment",
                startTime, timer.Elapsed, success: false
            );
            throw;
        }
    }
}

🔹 Query Telemetry với Kusto (KQL)

Kusto
// Top 10 slowest requests trong 24h
requests
| where timestamp > ago(24h)
| top 10 by duration desc
| project timestamp, name, duration, resultCode

// Exception count theo type
exceptions
| where timestamp > ago(7d)
| summarize count() by type
| order by count_ desc

// Availability test results
availabilityResults
| where timestamp > ago(24h)
| summarize
    successRate = countif(success == 1) * 100.0 / count()
    by name, location
| order by successRate asc
