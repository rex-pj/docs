Azure Container Registry (ACR)

Private Docker registry trên Azure — lưu trữ, quản lý và build container images

🔷 1. Tổng Quan ACR

Azure Container Registry (ACR) là private Docker-compatible registry cho phép lưu trữ và quản lý container images và related artifacts (Helm charts, OCI artifacts). ACR tích hợp với Azure pipelines và CI/CD workflows.

🔹 Các Trường Hợp Sử Dụng

  • Scalable orchestration systems — Kubernetes (AKS), Docker Swarm, Service Fabric
  • Azure services — ACI, App Service, Batch, Service Fabric
  • CI/CD — Azure DevOps, GitHub Actions, Jenkins
  • Build pipelines — Build, test, push images từ source code

🔷 2. Service Tiers

📌 Chọn Tier
  • Basic — Dev/learning, thấp throughput
  • Standard — Production hầu hết workloads
  • Premium — Geo-replication, Private Link, high throughput

🔷 3. Thao Tác Cơ Bản với ACR

🔹 Tạo Registry và Push Image

CLI
# Tạo resource group
az group create --name myRG --location eastus

# Tạo ACR (Basic tier)
az acr create \
    --resource-group myRG \
    --name myregistry \
    --sku Basic

# Login vào ACR
az acr login --name myregistry

# Tag image với ACR login server
docker tag myapp:latest myregistry.azurecr.io/myapp:v1.0

# Push image
docker push myregistry.azurecr.io/myapp:v1.0

# List images trong registry
az acr repository list --name myregistry --output table

# Xem tags của image
az acr repository show-tags \
    --name myregistry \
    --repository myapp \
    --output table

🔷 4. ACR Tasks — Build Images trên Cloud

ACR Tasks cho phép build container images trên Azure — không cần Docker cài local.

🔹 Quick Task — Build & Push ngay lập tức

CLI
# Build từ Dockerfile trong thư mục hiện tại
az acr build \
    --registry myregistry \
    --image myapp:v1.0 \
    .

# Build từ GitHub URL
az acr build \
    --registry myregistry \
    --image myapp:latest \
    https://github.com/myorg/myrepo.git

🔹 Automated Task — Build khi có code changes

CLI
# Tạo task tự động build khi push lên GitHub
az acr task create \
    --registry myregistry \
    --name buildtask \
    --image myapp:{{.Run.ID}} \
    --context https://github.com/myorg/myrepo.git \
    --file Dockerfile \
    --git-access-token <PAT>

🔹 Multi-step Tasks (YAML)

CLI
# acr-task.yaml
version: v1.1.0
steps:
  - build: -t $Registry/myapp:$ID -f Dockerfile .
  - push: ["$Registry/myapp:$ID"]
  - cmd: $Registry/myapp:$ID --test
CLI
az acr run --registry myregistry --file acr-task.yaml .

Azure Container Instances (ACI)

Chạy containers nhanh, đơn giản — không cần quản lý infrastructure

🔷 1. Tổng Quan ACI

ACI là cách nhanh nhất và đơn giản nhất để chạy containers trên Azure không cần quản lý VMs. Phù hợp với:

  • Simple applications và batch jobs
  • Task automation và CI/CD builds
  • Applications cần isolation bảo mật cao (Hyper-V isolation)
  • Event-driven tasks (kết hợp với Azure Functions)

🔹 Ưu Điểm ACI

Ưu ĐiểmMô Tả
NhanhSeconds để start container, không cần provision VMs
Linh hoạtChọn CPU, memory chính xác theo nhu cầu
Bảo mậtHypervisor-level isolation — mỗi container group có kernel riêng
Chi phíPer-second billing — chỉ trả khi container chạy
Linux & WindowsHỗ trợ cả Linux containers và Windows containers

🔷 2. Container Groups

Container Group là đơn vị triển khai cơ bản của ACI — tương tự như Kubernetes Pod. Một Container Group chứa một hoặc nhiều containers dùng chung:

  • Cùng host machine
  • Cùng lifecycle (start/stop cùng nhau)
  • Cùng local network (giao tiếp qua localhost)
  • Cùng storage volumes
📦 Container 1: web-app
Image: nginx:latest
Port: 80 (public)
CPU: 1.0 · Memory: 1.5 GB
External IP: 20.xxx.xxx.xxx
🔍 Container 2: log-sidecar
Image: fluentd:latest
Port: (no public port)
CPU: 0.5 · Memory: 0.5 GB
Giao tiếp qua localhost (shared network)
💾 Shared Storage Volume (Azure Files Mount: /var/log)
Linux: ✅ Multi-container groups hỗ trợ
Windows: ✅ Single container only
Cả 2 containers dùng chung storage volume · lifecycle · network

🔹 Triển Khai Container Group

CLI
# Simple container (1 container)
az container create \
    --resource-group myRG \
    --name mycontainer \
    --image mcr.microsoft.com/azuredocs/aci-helloworld \
    --dns-name-label aci-demo \
    --ports 80
CLI
# Multi-container YAML deployment
# multi-container.yaml
apiVersion: 2019-12-01
location: eastus
name: myContainerGroup
properties:
  containers:
  - name: aci-tutorial-app
    properties:
      image: mcr.microsoft.com/azuredocs/aci-helloworld
      resources:
        requests:
          cpu: 1
          memoryInGb: 1.5
      ports:
      - port: 80
  - name: aci-tutorial-sidecar
    properties:
      image: mcr.microsoft.com/azuredocs/aci-tutorial-sidecar
      resources:
        requests:
          cpu: 0.5
          memoryInGb: 0.5
  osType: Linux
  ipAddress:
    type: Public
    ports:
    - protocol: tcp
      port: 80
tags: null
type: Microsoft.ContainerInstance/containerGroups
CLI
az container create --resource-group myRG --file multi-container.yaml

🔷 3. Restart Policies, Environment Variables & Volumes

🔹 Restart Policies

PolicyMô TảDùng Khi
Always (mặc định)Luôn restart container khi dừngLong-running apps (web servers)
NeverKhông bao giờ restartOne-time jobs
OnFailureRestart chỉ khi exit với non-zero codeBatch processing với error handling
CLI
az container create \
    --resource-group myRG \
    --name myjob \
    --image myregistry.azurecr.io/mytask \
    --restart-policy OnFailure

🔹 Environment Variables

CLI
# Plain text env vars
az container create \
    --resource-group myRG \
    --name mycontainer \
    --image myapp \
    --environment-variables \
        DB_HOST=myserver.database.windows.net \
        DB_PORT=1433

# Secure env vars (không hiển thị trong logs)
az container create \
    --resource-group myRG \
    --name mycontainer \
    --image myapp \
    --secure-environment-variables \
        DB_PASSWORD=MySuperSecretPassword

🔹 Mount Azure File Share

CLI
az container create \
    --resource-group myRG \
    --name mycontainer \
    --image mcr.microsoft.com/azuredocs/aci-tutorial-app \
    --azure-file-volume-account-name mystorageaccount \
    --azure-file-volume-account-key "$STORAGE_KEY" \
    --azure-file-volume-share-name myshare \
    --azure-file-volume-mount-path /var/www/html

Azure Container Apps

Serverless microservices platform — scale to zero, Dapr, built-in ingress

🔷 1. Kiến Trúc Container Apps

🌍 Container Apps Environment — Shared VNet, Log Analytics, Dapr
Container App A (API service)
Revision 1 → 10% traffic
Revision 2 → 90% traffic
Replica 1 + Replica 2 (scale out)
Container App B (Background job)
Scale: 0 → 10 replicas
KEDA trigger (Queue, HTTP, CPU)
Scale to zero khi idle
⚙️ Platform Features
Dapr Sidecar — state, pub/sub, service call
Service Discovery — tự động trong environment
Managed Certs — TLS tự động
🌐 Ingress
External — accessible từ Internet
Internal — chỉ trong environment
Users/Clients → Internet/VNet → Environment

🔹 Các Thành Phần

Thành PhầnMô Tả
EnvironmentBoundary cô lập cho Container Apps. Apps trong cùng environment dùng chung VNet, Log Analytics workspace.
Container AppỨng dụng container với một hoặc nhiều containers. Có thể có nhiều revisions.
RevisionImmutable snapshot của Container App. Dùng cho blue/green deployments và A/B testing.
ReplicaInstance của revision. Scale in/out tự động.

🔷 2. Revisions, Secrets & Scaling

🔹 Revisions — Immutable Deployments

myapi--abc123 (90% traffic)
  • Container: myapp:v2 (stable)
  • Active revision — nhận đa số traffic
myapi--def456 (10% traffic)
  • Container: myapp:v3-beta
  • A/B Testing — test với 10% users
  • Blue/Green: swap 100% khi ready
CLI
# Deploy Container App
az containerapp create \
    --name myapi \
    --resource-group myRG \
    --environment myEnvironment \
    --image myregistry.azurecr.io/myapi:v2 \
    --target-port 3000 \
    --ingress external \
    --min-replicas 1 \
    --max-replicas 10

# Update image (tạo revision mới)
az containerapp update \
    --name myapi \
    --resource-group myRG \
    --image myregistry.azurecr.io/myapi:v3

# Traffic routing giữa revisions
az containerapp ingress traffic set \
    --name myapi \
    --resource-group myRG \
    --revision-weight myapi--abc123=90 myapi--def456=10

🔹 Secrets trong Container Apps

CLI
# Thêm secret
az containerapp secret set \
    --name myapi \
    --resource-group myRG \
    --secrets "db-password=MySuperSecret"

# Reference secret trong environment variable
az containerapp update \
    --name myapi \
    --resource-group myRG \
    --set-env-vars "DB_PASSWORD=secretref:db-password"

🔹 Scaling Rules

Container Apps hỗ trợ scaling dựa trên KEDA (Kubernetes Event-driven Autoscaling):

TriggerMô Tả
HTTPScale dựa trên số concurrent HTTP requests
CPUScale khi CPU vượt ngưỡng
MemoryScale khi memory vượt ngưỡng
Azure QueueScale dựa trên độ dài Azure Queue Storage
Service BusScale dựa trên Service Bus queue length
Custom KEDA scalersKafka, Redis, SQL, v.v.
CLI
# Scale dựa trên HTTP requests
az containerapp update \
    --name myapi \
    --resource-group myRG \
    --min-replicas 0 \     # Scale to zero!
    --max-replicas 20 \
    --scale-rule-name http-rule \
    --scale-rule-type http \
    --scale-rule-http-concurrency 50  # 50 concurrent requests per replica

🔷 3. Dapr Integration

Container Apps tích hợp Dapr (Distributed Application Runtime) — framework giúp xây dựng microservices dễ dàng hơn với:

🔹 Dapr Building Blocks

Building BlockMô Tả
Service-to-Service InvocationGọi các services khác qua Dapr sidecar (service discovery tự động)
State ManagementLưu trữ state với Redis, Cosmos DB, v.v. — abstracted API
Pub/Sub MessagingPublish và subscribe events qua Service Bus, Event Hubs, Redis
BindingsKết nối với external services (Storage, Twilio, SMTP, v.v.)
ActorsStateful, concurrent actor model
SecretsĐọc secrets từ Key Vault, environment variables
🚀 Your App (Port 3000)
GET /orders · POST /pay
Gọi Dapr qua HTTP: localhost:3500
Không cần SDK — chỉ HTTP calls
⚡ Dapr Sidecar (Port 3500)
State mgmt → Redis, Cosmos DB
Pub/Sub → Service Bus, Event Hubs
Service call → service discovery tự động
Bindings → Storage, Twilio, SMTP
☁️ External Services
🔴 Redis — State storage
🔵 Service Bus — Pub/Sub messaging
🔑 Key Vault — Secrets
CLI
# Enable Dapr cho Container App
az containerapp dapr enable \
    --name myapi \
    --resource-group myRG \
    --dapr-app-id myapi \
    --dapr-app-port 3000

# Gọi service khác qua Dapr (trong code):
# GET http://localhost:3500/v1.0/invoke/orders-service/method/orders

🔹 Ingress — Expose Container Apps

CLI
# External ingress (accessible từ internet)
az containerapp create \
    --name myapi \
    --ingress external \
    --target-port 3000

# Internal ingress (chỉ accessible trong environment)
az containerapp create \
    --name myworker \
    --ingress internal \
    --target-port 5000

# URL format:
# External: https://myapi.<unique-id>.<region>.azurecontainerapps.io
# Internal: https://myworker.<env-name>.<region>.azurecontainerapps.io
