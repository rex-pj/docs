📌 Chọn Tier
  • Basic — Dev/test, chỉ cần queues đơn giản
  • Standard — Dev/test và production nhẹ, cần topics
  • Premium — Production với yêu cầu cao về performance, compliance, message size lớn

🔷 2. Tính Năng Nâng Cao

Tính NăngMô Tả
Message SessionsTạo FIFO guarantee. Exclusive, ordered handling của unbounded sequences of related messages.
AutoforwardingChaining queue/subscription sang queue/topic khác trong cùng namespace.
Dead-letter Queue (DLQ)Chứa messages không thể deliver đến receiver. Có thể inspect và remove messages từ DLQ.
Scheduled DeliverySubmit messages để xử lý vào thời điểm cụ thể trong tương lai.
Message DeferralClient defer retrieval đến sau. Message vẫn trong queue nhưng được "set aside".
TransactionsGroup nhiều operations vào execution scope — atomic (all or nothing).
Filtering & ActionsSubscribers định nghĩa rules để chọn messages từ topic muốn nhận.
Duplicate DetectionSender resend cùng message → queue/topic tự loại bỏ duplicates.
Geo-disaster RecoveryKhi Azure region downtime, tiếp tục xử lý ở region khác.

🔷 3. Queues, Topics & Subscriptions

📨 Queue (Point-to-Point)
1
Sender gửi message vào Queue
2
[Queue] lưu trữ, đảm bảo delivery
3
1 Receiver nhận và xử lý
1 message = 1 receiver duy nhất
Dùng cho: task queues, work distribution
📡 Topic/Subscription (Pub-Sub)
1
Publisher gửi message đến Topic
2
Topic fan-out sang các Subscriptions
3
Sub A → Receiver A · Sub B → Receiver B · Sub C → Receiver C
1 message = nhiều receivers
Filter: SQL Filter · Correlation Filter · True/False Filter
💀 Dead-Letter Queue (DLQ)

Messages thất bại được chuyển vào DLQ để inspect & reprocess. Nguyên nhân: MaxDeliveryCount exceeded · TTL expired · filter evaluation error · explicit dead-lettering.

🔷 4. Code: Gửi & Nhận Messages (.NET)

CLI
dotnet add package Azure.Messaging.ServiceBus

🔹 Gửi Messages đến Queue

C#
using Azure.Messaging.ServiceBus;

string connectionString = "Endpoint=sb://...;SharedAccessKeyName=...;SharedAccessKey=...";
string queueName = "orders";

await using var client = new ServiceBusClient(connectionString);
ServiceBusSender sender = client.CreateSender(queueName);

// Gửi 1 message
ServiceBusMessage message = new ServiceBusMessage("Order #12345")
{
    ContentType = "application/json",
    Subject = "OrderCreated",
    MessageId = Guid.NewGuid().ToString(),
    TimeToLive = TimeSpan.FromHours(24)
};
await sender.SendMessageAsync(message);

// Gửi batch messages (efficient)
using ServiceBusMessageBatch messageBatch = await sender.CreateMessageBatchAsync();
for (int i = 0; i < 10; i++)
{
    var msg = new ServiceBusMessage($"Order #{i}");
    if (!messageBatch.TryAddMessage(msg))
        throw new Exception("Batch quá lớn");
}
await sender.SendMessagesAsync(messageBatch);

🔹 Nhận Messages từ Queue

C#
ServiceBusProcessor processor = client.CreateProcessor(queueName, new ServiceBusProcessorOptions
{
    AutoCompleteMessages = false,
    MaxConcurrentCalls = 4
});

processor.ProcessMessageAsync += async args =>
{
    string body = args.Message.Body.ToString();
    Console.WriteLine($"Received: {body}");

    // Xử lý message thành công → Complete (xóa khỏi queue)
    await args.CompleteMessageAsync(args.Message);

    // Nếu xử lý thất bại → Abandon (đưa lại vào queue)
    // await args.AbandonMessageAsync(args.Message);

    // Gửi vào Dead-letter queue
    // await args.DeadLetterMessageAsync(args.Message, "reason", "description");
};

processor.ProcessErrorAsync += args =>
{
    Console.WriteLine($"Error: {args.Exception}");
    return Task.CompletedTask;
};

await processor.StartProcessingAsync();
Console.ReadKey();
await processor.StopProcessingAsync();

Azure Queue Storage

Simple, scalable message queue — lưu trữ lượng lớn messages

🔷 1. Tổng Quan Queue Storage

Azure Queue Storage là dịch vụ lưu trữ lượng lớn messages. Bạn có thể truy cập messages từ bất kỳ đâu trên thế giới qua HTTP hoặc HTTPS. Một queue message có thể lên đến 64 KB. Một queue có thể chứa hàng triệu messages, tới giới hạn total capacity của storage account.

🔹 Các Thành Phần

🗄️ Storage Account → Queue Service
Queue: "orders" — chứa nhiều Messages
Queue: "notifications" — chứa Messages
Message max 64 KB · TTL ≤ 7 ngày
URL: https://<account>.queue.core.windows.net/<queue>
🔄 Typical Pattern
Producer (Worker Role / App) — enqueue messages
[Queue] — buffer, decouples producer/consumer
Consumer (Worker) — dequeue, process, delete
At-least-once delivery qua visibility timeout

🔹 Cơ Chế Visibility Timeout

Consumer lấy message
↓
Message INVISIBLE trong Visibility Timeout period
↓
✅ Xử lý thành công
↓
DELETE message
Xóa vĩnh viễn khỏi queue
⏰ Timeout hết hạn
↓
Message VISIBLE lại
Consumer khác có thể lấy được
Cơ chế at-least-once delivery của Azure Queue Storage

🔷 2. Code: Azure Queue Storage (.NET)

CLI
dotnet add package Azure.Storage.Queues
C#
using Azure.Storage.Queues;
using Azure.Storage.Queues.Models;

string connectionString = "DefaultEndpointsProtocol=https;AccountName=...;AccountKey=...";
string queueName = "orders";

// Tạo QueueClient
QueueClient queueClient = new QueueClient(connectionString, queueName);

// Tạo queue nếu chưa có
await queueClient.CreateIfNotExistsAsync();

// ── GỬI MESSAGE ──
await queueClient.SendMessageAsync("Hello from Queue Storage!");

// Gửi với TTL và visibility delay
await queueClient.SendMessageAsync(
    messageText: "Delayed message",
    visibilityTimeout: TimeSpan.FromSeconds(30),  // Ẩn 30s trước khi visible
    timeToLive: TimeSpan.FromHours(1)              // Tự xóa sau 1h
);

// ── XEM MESSAGE (không xóa) ──
PeekedMessage[] peekedMessages = await queueClient.PeekMessagesAsync(maxMessages: 5);
foreach (var msg in peekedMessages)
{
    Console.WriteLine($"Peeked: {msg.MessageText}");
}

// ── NHẬN VÀ XỬ LÝ MESSAGE ──
QueueMessage[] messages = await queueClient.ReceiveMessagesAsync(maxMessages: 10);
foreach (QueueMessage message in messages)
{
    Console.WriteLine($"Processing: {message.MessageText}");

    try
    {
        // Xử lý message...
        await ProcessOrderAsync(message.MessageText);

        // Xóa message sau khi xử lý thành công
        await queueClient.DeleteMessageAsync(message.MessageId, message.PopReceipt);
    }
    catch (Exception ex)
    {
        // Không xóa → message sẽ visible lại sau visibility timeout
        Console.WriteLine($"Failed: {ex.Message}");

        // Hoặc update visibility timeout để xử lý sau
        await queueClient.UpdateMessageAsync(
            message.MessageId,
            message.PopReceipt,
            message.MessageText,
            TimeSpan.FromMinutes(5)  // Ẩn thêm 5 phút
        );
    }
}

// ── THỐNG KÊ QUEUE ──
QueueProperties properties = await queueClient.GetPropertiesAsync();
Console.WriteLine($"Messages in queue: {properties.ApproximateMessagesCount}");

// ── XÓA TẤT CẢ MESSAGES ──
await queueClient.ClearMessagesAsync();

🔹 Encode Messages (Base64)

⚠️ Message Encoding

Mặc định, QueueClient mới không encode messages. Nếu cần tương thích với code cũ hoặc Azure SDK v11, dùng QueueMessageEncoding.Base64:

JavaScript
var queueClient = new QueueClient(connectionString, queueName, new QueueClientOptions
{
    MessageEncoding = QueueMessageEncoding.Base64
});

Tóm Tắt — Điểm Quan Trọng Cho Kỳ Thi

Tóm Tắt — Điểm Quan Trọng Cho Kỳ Thi
💬 GIẢI PHÁP MESSAGING — TÓM TẮT NHANH
🚌 Service Bus
  • AMQP 1.0 protocol (+ HTTP/REST, JMS 2.0 ở Premium)
  • 3 tiers: Basic (queues only) | Standard | Premium (HSM, 100MB)
  • Message sessions = FIFO guarantee
  • Dead-letter queue (DLQ) tích hợp sẵn
  • Topics + Subscriptions = publish-subscribe
  • Max message: 256KB (Standard), 100MB (Premium)
📬 Queue Storage
  • Simple HTTP/HTTPS REST API — không cần SDK
  • Max message size: 64 KB
  • Visibility timeout: at-least-once delivery
  • TTL tối đa: 7 ngày
  • Không có FIFO guarantee (approximate ordering)
  • Dùng khi cần lưu >80GB messages hoặc audit trail
🤔 Khi Nào Chọn Gì?
  • Service Bus: FIFO, pub/sub, DLQ, enterprise, msg >64KB
  • Queue Storage: queue >80GB, audit log, giải pháp đơn giản/rẻ
